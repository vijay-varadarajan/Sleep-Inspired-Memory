# Sleep-inspired memory consolidation for LLMs and AI Agents

## Abstract (150 words)
This paper presents a code-grounded research draft for a sleep-inspired memory consolidation framework implemented for LLM-based conversational agents. The repository operationalizes a biologically motivated memory hierarchy with episodic traces, consolidated summaries, and schema-level abstractions, connected through an offline “sleep” cycle. The system integrates prioritized replay, generative compression, schema induction, and decay, and evaluates behavior across persona-centric and long-horizon conversational benchmarks. We analyze the architecture end to end, including preprocessing pipelines, multi-method baselines, dataset-aware memory configuration, and benchmark orchestration for PersonaMem, PersonaChat, and LOCOMO. The implementation emphasizes a unified agent interface so memory and non-memory methods remain directly comparable under shared evaluation protocols. In addition, the project includes a lightweight web interface for interactive inspection of memory behavior. This draft synthesizes the current implementation into a professional research narrative, identifies methodological and engineering gaps, and proposes a structured framework for future research-grade iterations, reliable reproducibility, and deployment-oriented validation planning.

## Keywords (6-10)

## Introduction

### Background (150 words)
Large language models can produce fluent responses but often struggle to preserve long-range, user-specific, and cross-session consistency when interaction histories become complex. Cognitive science offers a useful analogy: biological systems separate immediate episodic encoding from slower consolidation processes that form stable abstractions over time. This repository translates that idea into software by separating memory into episodic, consolidated, and schema layers, then periodically invoking a sleep-like transformation process. The implementation combines memory storage, retrieval, compression, and decay with dataset-specific policies that adapt behavior to conversational domains. Unlike static context-window methods, the system continuously rewrites memory structure to balance retention and forgetting. The codebase also includes baseline methods that isolate contributions of retrieval, episodic persistence, and summarization, enabling comparative analysis against the full sleep-enabled design. As a result, the project provides an executable testbed for studying whether biologically inspired consolidation can improve robust, context-aware behavior in LLM agents beyond immediate prompt conditioning.

### Motivation & scope (100 words)
The motivation is to move beyond short-context prompting toward durable, adaptive memory for agents that must reason across sessions, preferences, and evidence-linked facts. The scope of this draft is the full repository: core agent logic, memory stores, sleep pipeline, baseline implementations, dataset preprocessing and postprocessing, evaluation routines, and web-facing interaction tooling. We do not report empirical outcomes here; all numeric findings are intentionally omitted. Instead, we formalize the implemented design, map it to research components, identify current limitations visible in code-level decisions, and define a structured framework that can support reproducible benchmarking, ablation studies, future methodological extensions, and practical implementation guidance.

## Problem statement (60 words)
Current LLM agents in multi-session settings face instability in long-term recall, factual grounding, and preference continuity. The codebase addresses this by introducing sleep-inspired consolidation, yet still requires a unified research framing that connects architecture, datasets, methods, and evaluation into a publication-ready narrative. This draft provides that framing without inserting outcome claims, benchmark values, or comparative performance assertions at this stage.

## Gaps identified (80 words / 5-6 gaps)
1. Retrieval ranking relies heavily on concept overlap, with limited temporal and speaker constraints.
2. Compression is LLM-mediated and can lose fine-grained factual details.
3. Schema abstraction may over-generalize when strict evidence grounding is required.
4. Sleep scheduling is mostly fixed rather than dynamically workload-adaptive.
5. Judge-based evaluation introduces model-dependent scoring variability across datasets.
6. End-to-end tests for full consolidation-to-evaluation loops are comparatively limited, reducing confidence in deployment transferability.

## Proposed framework

### High level system overview (100 words)
The framework is organized as a modular memory-agent stack. A unified interaction interface captures user turns, retrieves candidate memories, and generates responses through an LLM. Incoming interactions are stored as episodic records with metadata. During scheduled sleep cycles, selected episodes are replayed, compressed into consolidated memories, and clustered into schemas representing higher-level regularities. Retrieval at inference time blends episodic recency, consolidated concepts, and schema abstractions into prompt-ready context. Dataset-aware configuration governs salience, evidence sensitivity, decay, and replay behavior. A benchmark layer orchestrates baselines and sleep-enabled runs under common protocols, while preprocessing and postprocessing layers maintain standardized dataset schemas and result artifacts for analysis.

### Conceptual pipeline (100 words)
The conceptual pipeline proceeds in six stages: (1) dataset ingestion and normalization into a common sample format; (2) online interaction where prompts are enriched by retrieved memory context; (3) episodic encoding of user-agent exchanges with metadata; (4) offline sleep processing with prioritized replay, generative compression, schema induction, and decay; (5) method-level benchmarking across vanilla, retrieval, episodic, summarization, and sleep-enabled variants; and (6) export of structured artifacts for table generation and downstream interpretation. ## Methodology

### Datasets descriptions (150 words)
The implementation supports three benchmark families with a shared runtime format. PersonaMem contributes preference-centered, persona-conditioned question answering derived from structured records and conversation snippets. PersonaChat contributes dialogue-oriented persona consistency data, emphasizing conversational continuity and turn-level adaptation. LOCOMO contributes long-horizon, session-linked conversational reasoning with evidence-centric question-answer structures requiring stronger grounding discipline. Dataset-specific preprocessing scripts map heterogeneous raw schemas into a unified representation containing query, target answer, distractors, persona text, and contextual snippets. Additional grouping by persona identity preserves multi-turn structure for continuity evaluation and sleep-cycle probing. Postprocessing scripts transform run outputs into standardized table-oriented artifacts so methods are compared under the same reporting schema. This design allows one evaluation backend to operate across datasets while still enabling dataset-aware policy differences in replay weighting, retrieval priorities, and factuality controls. Quantitative dataset-level outcomes are intentionally omitted and marked as placeholders in this draft for transparent, future population.

### Agent and memory architecture
The main agent composes three memory stores: episodic memory for raw interaction traces, consolidated memory for compressed summaries linked to source episodes, and schema memory for higher-order abstractions. The interaction loop optionally retrieves relevant memory context, builds a system prompt, calls the LLM, and writes new episodic entries. Consolidation metadata and access patterns are tracked to influence future replay and retrieval decisions. A configuration layer resolves dataset-aware policies that modulate salience weighting, retrieval emphasis, decay behavior, and evidence requirements. This architecture keeps API compatibility across methods while enabling progressive memory sophistication. The web module provides an operational front end for interactive exploration of the same backend memory pathways.

### Sleep cycle and replay policy
Sleep processing is implemented as a multi-phase orchestration routine. First, unconsolidated episodes are prioritized for replay using recency, importance, novelty, and access-informed signals. Second, selected episodes are compressed through LLM prompting into structured consolidated memories, with concept extraction and confidence-like metadata. Third, schema induction scans consolidated memories for recurrent patterns and forms abstractions with linked examples. Fourth, decay removes or deprioritizes stale low-value episodic traces, approximating forgetting. Statistics from each stage are returned to support instrumentation and analysis. The policy can be tuned by dataset configuration, and ablation flags can disable specific mechanisms such as schema use or replay prioritization for controlled comparative experimentation.

### Retrieval, context assembly, and response generation
At inference time, the agent constructs memory context by combining signals from episodic recency, consolidated concept matches, and schema-level associations. Concept extraction from incoming queries drives lookup across long-term memory layers, while retrieval outputs are formatted into compact prompt context for the LLM. The system keeps a bounded conversation history to preserve immediate dialogue flow without unbounded prompt growth. This approach balances short-term coherence with long-term memory reuse. Baseline methods in the same repository provide controlled alternatives: no-memory generation, vector retrieval, episodic-only recall, and lightweight summarization. The comparative structure isolates which parts of sleep-inspired consolidation contribute beyond standard retrieval or persistent raw history.

### Evaluation protocol and benchmark instrumentation
Evaluation is organized into two methodological groups. Table-oriented task metrics assess long-horizon QA alignment, multi-session continuity, hallucination tendencies, and additional utility or retrieval diagnostics. Cognitive-style probes compare behavior before and after consolidation, including delayed recall, cue-based recall, cross-episode integration, and schema utilization. A common evaluator interface is reused across datasets and methods to preserve comparability. Runner scripts coordinate persona-wise sampling, method execution, exception handling, and artifact export. Results are serialized into structured JSON and CSV outputs for downstream analysis. ### Implementation workflow, reproducibility, and limitations
The codebase is organized for reproducible experimentation through explicit preprocessing, method factories, configurable memory policies, and deterministic output serialization paths. Baseline and sleep-enabled agents share a common interaction contract, which simplifies controlled comparisons and ablation studies. Existing tests cover memory store behavior, replay utilities, and benchmark smoke checks, though broader end-to-end validation remains a growth area. External model calls introduce dependency on API availability and model behavior, making strict reproducibility sensitive to runtime environment and provider updates. Repository documentation captures workflow assumptions and known dataset-specific limitations; this draft aligns those implementation realities into a research narrative while reserving all empirical claims for future result-populated versions.
